Routes

Some data types may be outdated, it is recommended to see the latest GitHub documentation.

Query parameters:

  • ID: Unique identifier.
  • Name: Class name, example: Warlock.
  • Title: Name of the guide.
  • Branch: Name of the branch class.
  • Difficulty: Percentage of class difficulty.
  • Tutorial: URL YouTube Video.
  • Description: Description of the guide.
  • Role: Role of the guide.

Types
All ID

The All type accepts both an empty value and an ID. For example:

  • /api/guides
  • /api/guides/:ID

The ID type only accepts values with an identifier. For example:

  • /api/guides/consumables/:ID
